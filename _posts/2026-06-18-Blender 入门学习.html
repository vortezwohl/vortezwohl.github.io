Blender 入门学习

这篇文章把 Blender 新手最常遇到的几个问题放在一条线里回答：怎么下载、是否免费、该怎么开始建模、模型导出应该选什么格式，以及 Blender 的 Python API 该从哪里入手。正文以官方站点和官方文档为主，覆盖基础上手与自动化两个方向。

Blender 本体免费且开源，个人学习和商业项目都可以使用。

核心结论 2

核心结论 3

核心结论 4

一、先看结论

适合谁

适合准备系统学习 3D 基础的人，也适合要给游戏引擎、网页 3D 项目或渲染流程生产资产的人。

最小闭环

安装 Blender，做一个简单模型，导出成目标格式，再去目标环境里验证比例、朝向和材质。

Python 切入点

先分清 bpy.data、bpy.ops、bpy.context，再进入 bmesh 和 Add-on 结构。

如果你现在的目标只是“先做出第一个能导出的模型”，不要一开始跳进复杂角色、骨骼和雕刻流程。

1. Blender 是什么

Blender 是一套免费、开源的 3D 创作软件，覆盖建模、雕刻、动画、渲染、合成和视频剪辑等流程。它不仅能做静态模型，也能做角色、场景、材质和动画资产。

从实际用途看，Blender 适合三类场景：

  • 学习 3D 基础与建模流程；
  • 制作可交付给游戏引擎或网页项目的资产；
  • 完整的内容创作流程，包括动画和渲染。

2. 是否免费

结论很明确：Blender 本体是免费的，而且是开源软件。

  • 官方许可页说明 Blender 可以用于商业项目；
  • 它不是试用版，也不是订阅制；
  • 个人学习、独立制作、工作室商业项目都可以使用。

需要注意的是，免费指的是软件授权本身，不等于所有附带资源都自动免费。你在制作中使用的贴图、模型、字体、HDRI、音乐等素材，仍然要分别确认授权。

参考：

  • https://www.blender.org/about/license/

3. 如何下载

最稳妥的方式只有一个：从 Blender 官方网站下载。

下载入口

  • 官方下载页：https://www.blender.org/download/
  • LTS 版本页：https://www.blender.org/download/lts/
  • 历史版本页：https://www.blender.org/download/previous-versions/

下载建议

  • 如果你是新手，直接下载官方稳定版即可；
  • 如果你要做长期项目，优先选 LTS；
  • 如果你只是想尝试新功能，可以看实验版，但不建议拿来做稳定交付。

本次调研时看到的版本信息

  • 官方下载页当前显示稳定版为 Blender 5.1.2；
  • LTS 分支是 Blender 4.5 LTS；
  • 官方对 LTS 的定位是给长期项目使用的稳定版本，并提供官方支持。

常见安装建议

  • Windows：下载对应安装包，按默认流程安装；
  • macOS：下载 DMG，拖入 Applications；
  • Linux：优先用官方发布包或发行版仓库提供的版本。

4. 如何建模

Blender 的建模可以理解成：先搭大形，再加结构，再加细节。

新手最推荐的建模路线

  1. 添加一个基础体，例如 Cube、Cylinder 或 Sphere；
  2. 进入 Edit Mode；
  3. 用挤出、环切、缩放、移动做出大轮廓；
  4. 用倒角和细分修形；
  5. 最后再处理材质、UV 和导出。

最常用的操作

  • G：移动；
  • R：旋转；
  • S：缩放；
  • E：挤出；
  • Ctrl + R：环切；
  • Ctrl + B：倒角；
  • I：内插。

实际建模思路

  • 做杯子、桌子、箱子这类简单物体，最适合练基础；
  • 做角色时，优先做低模和大形，不要一开始就追细节；
  • 对称物体建议优先使用 Mirror 修饰器；
  • 想要更平滑的效果，可以配合 Subdivision Surface。

适合新手的练习顺序

  • 第 1 个模型：杯子；
  • 第 2 个模型：桌子；
  • 第 3 个模型：简单箱体道具；
  • 第 4 个模型：低模角色头部；
  • 第 5 个模型：带骨骼的完整角色。

5. 如何导出模型

Blender 支持很多导出格式，官方手册明确提到常见格式包括 ABC、USD、OBJ、FBX、PLY、STL 等。

参考：

  • https://docs.blender.org/manual/en/latest/files/import_export/index.html

5.1 glTF / GLB

适合：网页展示、实时渲染、Three.js、部分游戏流程。

优点：

  • 对材质和动画支持较好；
  • 适合现代实时渲染管线；
  • GLB 是二进制打包格式，便于分发。

建议：

  • 如果你的目标是网页或在线 3D 展示，优先用 GLB；
  • 如果要保留动画，优先检查导出设置里与动画相关的选项；
  • 导出前最好只选中需要的对象。

参考：

  • https://docs.blender.org/manual/en/latest/addons/import_export/scene_gltf2.html
  • https://threejs.org/docs/pages/GLTFLoader.html

5.2 FBX

适合：Unity、Unreal、DCC 软件之间交换角色、骨骼、复杂场景数据。

优点：

  • 对游戏管线很常见；
  • 适合静态模型和带骨骼的角色；
  • 兼容性在引擎工作流里比较成熟。

建议：

  • 导出前应用缩放和旋转；
  • 骨骼角色要注意层级干净、根骨明确；
  • 导出后到目标引擎里再核对比例和朝向。

参考：

  • https://docs.blender.org/manual/en/latest/addons/import_export/scene_fbx.html
  • https://docs.unity3d.com/6000.4/Documentation/Manual/FBXImporter-Model.html
  • https://dev.epicgames.com/documentation/unreal-engine/fbx-import-options-reference-in-unreal-engine

5.3 OBJ

适合：静态模型交换。

限制：

  • 不适合骨骼和动画；
  • 相机、灯光等信息也不适合依赖 OBJ 传递；
  • 更像是一个“够简单但功能有限”的通用几何格式。

参考：

  • https://docs.blender.org/manual/en/latest/files/import_export/obj.html

5.4 STL

适合：3D 打印和 CAD 交换。

特点：

  • 结构简单；
  • 通常只关注几何；
  • 对 3D 打印流程很实用。

参考：

  • https://docs.blender.org/manual/en/latest/files/import_export/stl.html

6. 不同目标平台怎么选

Three.js / 网页

  • 首选：GLB / glTF；
  • 原因：Three.js 官方就提供了 GLTFLoader；
  • 如果只是静态展示，GLB 通常最省心。

Unity

  • 首选：FBX；
  • 适合静态道具、角色和动画资产；
  • 导入后重点检查尺度、朝向和材质映射。

Unreal

  • 首选：FBX；
  • Unreal 的单位体系是厘米，导入时要特别注意比例转换；
  • 角色和静态网格都可以走 FBX 管线。

3D 打印

  • 首选：STL；
  • 目标是保证几何能被打印软件正确识别。

7. 一个最短实践路径

如果你现在想立刻上手，建议按这个顺序来：

  1. 安装 Blender；
  2. 熟悉视图和快捷键；
  3. 做一个杯子；
  4. 做一个桌子；
  5. 导出成 GLB 或 FBX；
  6. 在目标软件里验证模型是否正常；
  7. 再开始做更复杂的角色或场景。

8. 本次调研结论

  • Blender 是免费的、开源的 3D 创作软件；
  • 官方下载是最可靠的获取方式；
  • 新手建模应从基础体和网格编辑开始；
  • 导出时最常见的选择是 GLB、FBX、OBJ、STL；
  • 给 Three.js 优先用 GLB；
  • 给 Unity 和 Unreal 优先用 FBX；
  • 给 3D 打印优先用 STL。

10. Blender Python API 调研

Blender 的 Python 支持核心是 bpy，它把 Blender 的数据结构、上下文、操作器、网格编辑和插件系统都暴露给 Python。对于自动化、插件和程序化建模，这一层能力非常关键。

10.1 API 分层

可以把 API 理解成三层：

  • bpy.data：直接读写对象、网格、材质、场景等数据块；
  • bpy.ops：调用 Blender 内置操作器，行为更像在界面里点按钮；
  • bpy.context：读取当前上下文，例如活动对象、选中项和当前场景。

其中最重要的分界是：bpy.data 更稳定，bpy.ops 更依赖上下文。

参考：

  • https://docs.blender.org/api/current/index.html
  • https://docs.blender.org/api/current/info_overview.html

10.2 bpy.data

bpy.data 适合批量处理和精确修改。

import bpy for obj in bpy.data.objects: print(obj.name)

参考：

  • https://docs.blender.org/api/current/bpy.data.html

10.3 bpy.ops

bpy.ops 适合执行“像按钮一样”的流程型动作，例如添加基础体、渲染或导出。

import bpy bpy.ops.mesh.primitive_cube_add(location=(0, 0, 0))

但它对模式、选择状态和活动对象有要求。

参考：

  • https://docs.blender.org/api/current/bpy.ops.html

10.4 bpy.context

bpy.context 是只读上下文，内容会随着 UI 区域变化。很多 operator 失败，本质上是上下文不满足要求。

参考：

  • https://docs.blender.org/api/current/bpy.context.html

10.5 bmesh

做程序化建模或网格拓扑处理时，bmesh 往往比 bpy.ops 更合适。

import bpy import bmesh obj = bpy.context.active_object mesh = obj.data bm = bmesh.new() bm.from_mesh(mesh) for v in bm.verts: v.co.z += 1.0 bm.to_mesh(mesh) bm.free()

参考：

  • https://docs.blender.org/api/current/bmesh.html
  • https://docs.blender.org/api/current/bmesh.ops.html

10.6 mathutils

mathutils 提供 Vector、Matrix、Euler、Quaternion 等数学类型，适合坐标变换、旋转、射线检测和空间计算。

参考：

  • https://docs.blender.org/api/current/mathutils.html

10.7 插件骨架

做 Add-on 时，核心结构通常是：

  • Operator：执行动作；
  • Panel：提供按钮和界面；
  • PropertyGroup + bpy.props：定义自定义属性；
  • register() / unregister()：注册和注销类。

参考：

  • https://docs.blender.org/api/current/bpy.types.Operator.html
  • https://docs.blender.org/api/current/bpy.types.Panel.html
  • https://docs.blender.org/api/current/bpy.props.html
  • https://docs.blender.org/manual/en/latest/advanced/scripting/addon_tutorial.html

10.8 handlers / timers

如果脚本要在特定时机自动运行，可以看：

  • bpy.app.handlers：事件回调；
  • bpy.app.timers：定时回调。

参考：

  • https://docs.blender.org/api/current/bpy.app.handlers.html
  • https://docs.blender.org/api/current/bpy.app.timers.html

10.9 学习顺序

建议顺序是：

  1. bpy.data 和 bpy.context；
  2. bpy.ops 的上下文依赖；
  3. bpy.types.Operator、bpy.types.Panel、bpy.props；
  4. bmesh；
  5. mathutils；
  6. handlers 和 timers。

10.10 结论

  • Blender 的 Python API 很适合自动化、插件和程序化建模；
  • 新手先分清 bpy.data、bpy.ops 和 bpy.context；
  • 几何处理优先学 bmesh；
  • 数学和空间计算优先学 mathutils；
  • 插件开发的核心骨架是 Operator + Panel + props。
